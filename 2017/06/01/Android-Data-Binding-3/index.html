資料變動通知機制

透過 data binding 我們可以將 model 與 view 做綁定, 但是當 model 的資料變動時資料並不會自動更新 UI,
若要具有當資料變動時可以自動更新 UI 的能力, 我們可以使用以下3種不同的資料變動通知機制:

  • Observable Objects
  • Observable Fields
  • Observable Collections

Observable Objects

做法是, 將一個 model class 繼承 BaseObservable, 並在 getter 設定 Bindable annotation 以及在 setter 使用 notifyPropertyChanged() 方法, 這樣就可以監聽物件內屬性的變動

這邊的 BR 類別會在編譯時被產生, @Bindable annotation 會在 BR 內建立一個 entry, 在 setter 呼叫 notifyPropertyChanged 並傳入指定 entry

所以當我們用 setter 去修改物件內的屬性時, 被綁定的 UI 就會自動被更新

既然如此, 我們就在先前 Presenter 中透過 setter 修改 lastName

接著運行 APP, 點擊 firstName TextView 後可以發現 lastName 的資料被自動更新了
