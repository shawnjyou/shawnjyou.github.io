簡介

Data binding is a general technique that binds data sources from the provider and consumer together and synchronizes them.
Data binding 是一種通用技術, 用以綁定資料的提供對象以及使用對象並且同步。
- Wikipedia

在 Android 中使用 Data binding 的優點 :

  • 減少 findViewById() 樣板代碼 (Boilerplate code) 的使用
  • 分離 UI邏輯 與 控制邏輯
  • 更方便的方式同步 model & view 間的資料

基本使用

本篇使用 Google I/O 2015 發表的 Android Data Binding Library
該 Library 可以向前相容到 Android 2.1 (API level 7+)

View Layout

以往 layout 根節點通常是使用 ViewGroup 的子類 (e.g. RelationLayout 或 LinearLayout)
但若使用 Data Binding Library 時就必須用 <layout> 作為根節點

使用 @{} 語法將 TextView 對應到 model 中相對應的 property

綁定 View 與 Model

在預設的情況了, Data binding library 會基於 layout 檔名 : activity_main 來自動產生 Binding class : ActivityMainBinding

以往會以 setContentView(R.layout.activity_main) inflate layout, 再透過 findViewById()方法或 butterknife lib 來取得 view
而透過 Data Binding 取而代之的是使用 DataBindingUtil.setContentView(this, R.layout.activity_main) 產生對象給剛剛自動產生的 binding class, 再透過 binding class 的 set 方法綁定 model

此時就已經完成初步的資料綁定, 運行 APP 後可以看到兩個 TextView 上顯示了剛剛綁定相對應的資料

Custom binding class

剛才提到 binding class 會依照 layout filename 被自動產生, 但有時候我們可能想要自訂名稱或是將其放在指定的 package 底下方便管理

要達到這個效果, 只要在 layout xml 中, 設定 data 的 calss 屬性就可以自訂 binding class 名稱
若要指定存放的 package 只要在名稱前面加上 package name 即可

這樣 Library 就會幫我們產生名為 UserBinding 的 binding class 並放在指定的 package 底下
