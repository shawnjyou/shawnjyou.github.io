事件處理

開發時免不了要處理 view 元件的事件
使用 Android data binding library 提供兩種處理事件是方式

  • Method References
  • Listener Bindings

Method References

以往我們可以將 layout 的 android:onClick 參照到 Activity 裡 onClick() method, 而 Method References 只是將這個參照改到我們自己定義的 method 而已
這邊要注意的是自己定義的 method signature 與 listener method 的 signature 是一致的
Method References 是在編譯時被處理的, 所以當方法不存在或是 signature 不一致時, 在編譯時就可以得到錯誤。

假設我們想要處理用來顯示 firstName 這個 TextView 的 onClick event
新增一個 handler class, 並寫一個 onClickFirstName 方法

加入名為 handler 的 variable 指定到剛才建立的 MyHandler class
設定 android:onClick 屬性, 用 @{handler::onClickFirstName} 將其參照到 MyHandler 裡的 onClickFirstNam(View view)

將 handler 綁定到 binding class

運行 APP 點擊 TextView 就可以看到成功處理了 onClick event

Listener Bindings

是使用 Lambda expressions, 並會在事件觸發時被 evaluate,
與 Method References 不一樣的是, Listener bindings 可以運行隨意的資料綁定表示式,

這個 feature 只有在 Android Gradle Plugin for Gradle version 2.0 and later 才可以使用

在 Method References 中 method signature 要與 event listener signature 一致, 而 Listener bindings 則是表達式返回值型態要與 event listener 返回值型態一致 (除非是 return void)

新增一個 Presenter class, 並寫一個 onClickFirstName 方法

加入名為 presenter 的 variable 並對應到剛才建立的 Presenter class
設定表示式 android:onClick 屬性, 用 @{()->presenter.onClickFirstName(user)} 對應到 Presenter 裡的 onClickFirstName(User user)

將 presenter 綁定到 binding class

這樣 Listener Bindings 就完成了!

雖然 onClick 必須要傳入 View 作為參數, 但上面的表示式沒有傳入 View 作為參數是可以被允許的, 原因是 Listener Bindings 在 listener method 的參數方面提供兩種選擇 :

  • 忽略所有 listener method 的參數
  • 命名所有 listener method 的參數
